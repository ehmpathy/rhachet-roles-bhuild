for wishes

  - integration tests on all boundaries (e.g., with external systems, os, sdks, etc)
  - integration tests on end-to-end flows
  - acceptance tests on behavioral contracts

  - if probabilistic, whether or not there are open source examples of how to do this

- require.comprehensive: satisfies all criteria && wish

etc


checkout the patterns adopted here for how to structure rules (the directory + filepath structure)

https://github.com/ehmpathy/rhachet-roles-ehmpathy/blob/main/src/domain.roles/mechanic/briefs/practices

we want to declare these rule briefs

---

we also want to declare a composite skill which builds on `npx rhachet run --repo bhrain --skill review` in order to apply these rules against a given behavior

e.g.,


```sh

npx rhachet run --repo bhrain --skill review \
  --rules .agent/**/*behaver/**/criteria.*/rule.*.md \
  --paths .behavior/v2025_12_23.say-hello/2.criteria.md \
  --output .behavior/v2025_12_23.say-hello/2.criteria.md.[feedback].v1.[given].by_robot.md

npx rhachet run --repo bhrain --skill review \
  --rules .agent/**/*behaver/**/blueprint.*/rule.*.md \
  --paths .behavior/v2025_12_23.say-hello/3.3.blueprint.md \
  --output .behavior/v2025_12_23.say-hello/3.3.blueprint.md.[feedback].v1.[given].by_robot.md

npx rhachet run --repo bhrain --skill review \
  --rules .agent/**/*behaver/**/roadmap.*/rule.*.md \
  --paths .behavior/v2025_12_23.say-hello/3.3.roadmap.md \
  --output .behavior/v2025_12_23.say-hello/3.3.roadmap.md.[feedback].v1.[given].by_robot.md
```

and builders can invoke via

```sh
npx rhachet run --repo bhuild --skill review.behavior --of say-hello
```

where .behavior/v2025_12_23.say-hello/ is autodetected (just like review.deliverable does) based on --of say-hello

(per https://github.com/ehmpathy/rhachet-roles-bhrain )
