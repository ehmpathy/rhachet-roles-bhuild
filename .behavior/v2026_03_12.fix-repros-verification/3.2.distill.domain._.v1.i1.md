# distill: domain

## scope note

this behavior modifies **template files** that are emitted by `initBehaviorDir`, not runtime domain objects.

the "domain" here is:
- template file content (stone and guard files)
- template file structure (which files exist)

no new domain.objects, domain.operations, or access.daos are needed.

---

## relationships

```
initBehaviorDir

       ├─ 3.2.distill.repros.*.stone  [EXTEND]

```

---

## composition

the flow:
1. `initBehaviorDir` is called
2. templates are copied to `.behavior/{name}/`
3. mechanic drives through route
4. at repros stone: enhanced content guides journey sketches
5. at verification stone: content references repros, guard enforces checks

no runtime domain objects are instantiated — this is pure template content.
