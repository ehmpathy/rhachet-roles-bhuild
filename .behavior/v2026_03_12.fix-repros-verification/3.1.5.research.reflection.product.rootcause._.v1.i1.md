# reflection: root cause

## what is the symptom?

**observable problem:**
- mechanics complete the behavior route but PR review reveals absent journey tests
- whatever acceptance tests exist lack snapshot assertions
- repros stone results are not considered in verification

**how it manifests:**
- foreman discovers gaps at PR review time
- rework is required after the route is "complete"
- no visible proof of what the caller experiences

**when it occurs:**
- at PR review, after execution stone is complete
- repeatedly, across multiple behaviors

---

## 5 whys tree

### branch 1: journey tests absent

1. why? → mechanics don't create journey tests
2. why? → repros stone doesn't require journey test sketches
3. why? → repros stone only asks for a table of experiences, not structured sketches
4. why? → original design focused on "what to test" not "how to structure tests"
5. why? → **ROOT CAUSE: repros stone lacks explicit BDD structure guidance**

### branch 2: snapshots absent

1. why? → tests don't have snapshot assertions
2. why? → verification doesn't check for snapshots
3. why? → verification checklist has snapshot section but no guard enforces it
4. why? → no self-review prompt asks "do snapshots capture input/output?"
5. why? → **ROOT CAUSE: snapshot coverage lacks self-review enforcement**

### branch 3: repros not connected to verification

1. why? → verification doesn't reference repros artifact
2. why? → verification checklist is generic, not tied to prior stone
3. why? → original design treated stones as independent checkpoints
4. why? → route didn't emphasize traceability between stones
5. why? → **ROOT CAUSE: repros → verification linkage was never prescribed**

---

## root cause summary

| layer | issue | is root cause? |
|-------|-------|----------------|
| symptom | absent tests and snapshots at PR review | no |
| layer 1 | repros doesn't require structured sketches | partial |
| layer 2 | verification doesn't reference repros | partial |
| layer 3 | no guard enforcement for snapshots | partial |
| **root** | stone content and guards lack explicit prescription | **yes** |

---

## recommendation

**address root cause:** update stone content and add guard self-reviews

the fix addresses the root by:
1. repros stone: add explicit BDD structure guidance, input/output pairs, snapshot targets
2. repros guard: add self-reviews for critical paths, ergonomics, convention
3. verification stone: reference repros artifact explicitly
4. verification guard: add self-reviews that check repros was implemented

this is a durable fix — stone content and guards ensure future mechanics can't skip these steps.
