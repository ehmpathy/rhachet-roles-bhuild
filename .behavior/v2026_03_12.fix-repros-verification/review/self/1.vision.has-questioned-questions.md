### 3. for journeys without stdout/stderr, should we require some snapshot?

**can this be answered via logic?** yes

**analysis:**
- some behaviors are file-based (emit files, modify configs)
- for these, "snapshot" = file content snapshot
- the goal is visibility for reviewers — what does the output look like?
- for file-based: snapshot the file content
- for api-based: snapshot the response structure
- for no-output: may not need snapshot (pure side effect, verified by assertion)

**self-answer:** require snapshot for any **user-visible output**:
- stdout/stderr → snapshot
- emitted files → snapshot file content
- api responses → snapshot response structure
- pure side effects (db writes, etc) → no snapshot needed, verified by assertion

**does wisher need to confirm?** no — this is a reasonable interpretation of "demonstrate what callers would see."
