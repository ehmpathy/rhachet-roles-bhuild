# blackbox criteria: fix repros-verification connection

## usecase.1 = mechanic completes repros stone

```
given('[case1] mechanic at repros stone with vision declared')
  when('[t0] mechanic reads the repros stone')
    then('stone content guides them to sketch journey tests')
    then('stone content explains BDD given/when/then structure')
    then('stone content explains [tN] label convention')
      sothat('mechanic knows the expected format')
  when('[t1] mechanic writes journey test sketches')
    then('each journey has a step table: step | action | user sees')
    then('each step has input/output pair documented')
    then('snapshot targets are marked for each output')
      sothat('mechanic plans what to capture')
  when('[t2] mechanic identifies critical paths')
    then('happy paths are marked as critical')
    then('ergonomics are reviewed for each path')
      sothat('friction is caught before implementation')
  when('[t3] mechanic completes repros guard self-reviews')
    then('has-critical-paths-identified is prompted')
    then('has-ergonomics-reviewed is prompted')
    then('has-play-test-convention is prompted')
      sothat('omissions are caught before stone passes')

given('[case2] mechanic at repros stone with no visible output behavior')
  when('[t0] mechanic writes journey for file-based behavior')
    then('snapshot targets are file content instead of terminal output')
      sothat('all user-visible outputs are captured')

given('[case3] mechanic at repros stone with dynamic output')
  when('[t0] mechanic writes journey with timestamps or uuids')
    then('stone content guides them to plan masks')
      sothat('snapshots are stable')
```

## usecase.2 = mechanic completes verification stone

```
given('[case1] mechanic at verification stone after execution')
  when('[t0] mechanic reads the verification stone')
    then('stone references repros artifact explicitly')
    then('stone asks: for each journey in repros, point to test file')
    then('stone asks: for each test, verify snapshot assertions exist')
    then('stone asks: for each critical path, verify frictionless')
      sothat('mechanic connects verification to repros')
  when('[t1] mechanic fills behavior coverage table')
    then('table has column: journey from repros')
    then('table has column: test file')
    then('table has column: snapshots present')
    then('table has column: critical path frictionless')
    then('table has column: ergonomics validated')
      sothat('every repros promise is tracked')
  when('[t2] mechanic completes verification guard self-reviews')
    then('has-journey-tests-from-repros is prompted')
    then('has-snapshot-coverage is prompted')
    then('has-critical-paths-frictionless is prompted')
    then('has-ergonomics-validated is prompted')
    then('has-play-test-convention is prompted')
      sothat('omissions are caught before stone passes')

given('[case2] mechanic at verification with journey test absent')
  when('[t0] mechanic answers has-journey-tests-from-repros')
    then('self-review forces reflection on each journey from repros')
    then('mechanic realizes gap and goes back to add test')
      sothat('no journey is left unimplemented')

given('[case3] mechanic at verification with snapshot absent')
  when('[t0] mechanic answers has-snapshot-coverage')
    then('self-review asks if snapshots capture input/output')
    then('mechanic realizes gap and adds snapshot assertion')
      sothat('reviewer can see what user sees')
```

## usecase.3 = foreman reviews PR

```
given('[case1] foreman reviews PR with journey tests')
  when('[t0] foreman opens snapshot files')
    then('snapshots show what caller would see')
    then('snapshots match what was planned in repros')
      sothat('foreman has confidence in the experience')
  when('[t1] foreman reviews verification artifact')
    then('behavior coverage table references repros')
    then('every journey has test file and snapshot')
      sothat('foreman can trace from wish to proof')
```

## usecase.4 = play test convention

```
given('[case1] repo supports .play.test.ts directly')
  when('[t0] mechanic names journey test file')
    then('file is named *.play.test.ts')
      sothat('journey tests are distinct from unit tests')

given('[case2] repo does not support .play.test.ts')
  when('[t0] mechanic names journey test file')
    then('file is named *.play.integration.test.ts')
    or('file is named *.play.acceptance.test.ts')
      sothat('journey tests run with integration or acceptance suite')
```
