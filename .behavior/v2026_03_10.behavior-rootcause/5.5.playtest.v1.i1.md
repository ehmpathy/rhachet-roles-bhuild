# playtest: reflection research stones

## prerequisites

- repo checked out: `rhachet-roles-bhuild`
- on a feature branch (not main/master)
- `npm ci` complete
- `npm run build` complete

## happy paths

### 1. verify templates exist in dist

```sh
ls -la dist/domain.operations/behavior/init/templates/3.1.5.*.stone
```

**expected outcome:**
- shows 3 files:
  - `3.1.5.research.reflection.product.audience._.v1.stone`
  - `3.1.5.research.reflection.product.rootcause._.v1.stone`
  - `3.1.5.research.reflection.product.premortem._.v1.stone`

### 2. verify init.behavior creates reflection stones

```sh
git checkout -b playtest/reflection-stones-$(date +%s)
npx rhachet run --skill init.behavior --name playtest-feature
```

**expected outcome:**
- output shows `+ 3.1.5.research.reflection.product.audience._.v1.stone`
- output shows `+ 3.1.5.research.reflection.product.rootcause._.v1.stone`
- output shows `+ 3.1.5.research.reflection.product.premortem._.v1.stone`
- stones appear in correct position (after 3.1.4, before 3.2)

### 3. verify audience stone content

```sh
cat .behavior/v*playtest-feature/3.1.5.research.reflection.product.audience._.v1.stone
```

**expected outcome:**
- prompts for "who is the audience?"
- prompts for "why do they care?"
- prompts for "how much do they care?"
- prompts for "what do they care about?"
- prompts for "how will they experience it?"
- emit path ends with `.i1.md`

### 4. verify rootcause stone content

```sh
cat .behavior/v*playtest-feature/3.1.5.research.reflection.product.rootcause._.v1.stone
```

**expected outcome:**
- prompts for "what is the symptom?"
- prompts for "5 whys" with branches
- prompts for root cause summary
- prompts for recommendation
- emit path ends with `.i1.md`

### 5. verify premortem stone content

```sh
cat .behavior/v*playtest-feature/3.1.5.research.reflection.product.premortem._.v1.stone
```

**expected outcome:**
- prompts for failure imagination
- prompts for ignored signs
- prompts for wrong assumptions
- prompts for nightmare scenario
- prompts for mitigations
- emit path ends with `.i1.md`

## edgey paths

### extant behavior reuses templates

```sh
npx rhachet run --skill init.behavior --name playtest-feature
```

**expected behavior:**
- idempotent — does not error
- does not duplicate stones

### cleanup

```sh
git checkout -
git branch -D playtest/reflection-stones-*
```

## pass/fail criteria

- pass if:
  - all 3 reflection stones created by init.behavior
  - each stone has correct prompts per vision
  - stones appear in correct number position (3.1.5)
  - templates follow extant structure (.why, reference, sections, emit)

- fail if:
  - any template file not created
  - prompts do not match vision requirements
  - stones have wrong number prefix
  - emit paths are incorrect
