# blackbox criteria

## usecase.1 = mechanic completes reflection stones

```
given('a behavior route with reflection stone files')
  when('mechanic reaches 3.1.5.research.reflection.product.audience stone')
    then('stone prompts for audience analysis')
      sothat('mechanic identifies who the change serves')
    then('stone prompts: who is the audience?')
    then('stone prompts: why do they care?')
    then('stone prompts: how much do they care?')
    then('stone prompts: what do they care about?')
    then('stone prompts: how will they experience it?')

  when('mechanic reaches 3.1.5.research.reflection.product.rootcause stone')
    then('stone prompts for root cause analysis')
      sothat('mechanic drills to true cause, not symptoms')
    then('stone prompts: what is the symptom?')
    then('stone prompts: 5 whys with branches')
    then('stone prompts: which layer holds the root cause?')
    then('stone prompts: recommendation for which layer to address')

  when('mechanic reaches 3.1.5.research.reflection.product.premortem stone')
    then('stone prompts for failure analysis')
      sothat('mechanic surfaces risks before they materialize')
    then('stone prompts: imagine we shipped and it failed — why?')
    then('stone prompts: what obvious signs did we ignore?')
    then('stone prompts: what assumptions could turn out wrong?')
    then('stone prompts: what is the nightmare scenario?')
    then('stone prompts: what mitigations to consider?')
```

## usecase.2 = blueprint guard validates reflection coverage

```
given('a behavior route with reflection stones and blueprint guard')
  when('mechanic attempts to pass blueprint stone')
    then('self-review prompts: has-addressed-reflections')
      sothat('blueprint demonstrates it solved the right problem')
    then('self-review checks: does blueprint address the audience?')
    then('self-review checks: does blueprint address root cause?')
    then('self-review checks: does blueprint mitigate risks?')
    then('self-review checks: if any skipped, is that justified?')

  when('blueprint does not address reflection findings')
    then('self-review blocks passage')
      sothat('bandaid fixes do not slip through')

  when('blueprint explicitly justifies symptom treatment')
    then('self-review allows passage with justification')
      sothat('pragmatic scope limits are documented')
```

## usecase.3 = reflection stones appear in correct order

```
given('a behavior route with all research stones')
  when('mechanic follows route order')
    then('3.1.3 code research stones come first')
    then('3.1.4 factory research stones come second')
    then('3.1.5 reflection stones come third')
    then('3.2 distill stones come fourth')
    then('3.3 blueprint stones come fifth')
      sothat('reflections inform distillation and blueprint')
```

## usecase.4 = reflection stone templates are available

```
given('behavior init command')
  when('route is created with reflection stones')
    then('audience stone template exists')
    then('rootcause stone template exists')
    then('premortem stone template exists')
      sothat('mechanics have clear prompts for each reflection')
```

## edge cases

```
given('a behavior with simple obvious fix')
  when('mechanic completes reflection stones')
    then('brief "N/A — obvious" with justification is valid')
      sothat('overhead is proportional to complexity')

given('a behavior where root cause is out of scope')
  when('mechanic documents root cause in external system')
    then('justified symptom treatment is valid')
    then('follow-up is tracked via reference')
      sothat('scope limits are explicit, not hidden')

given('a behavior where premortem finds showstopper')
  when('mechanic surfaces critical risk')
    then('route allows pause for wisher input')
      sothat('major risks are addressed before blueprint')
```
