wish =

this wish has two parts

main ask = 

lets formalize the blueprint for workloops vs blueprint for codepaths

workloops = the workflow feedback loops required to build the codepaths
- test loops
- dev loops
- dev infra

all of the tools, processes, and feedback loops required to build code and verify it does the right thing

vs

codepaths = the code we construct in order to fulfill the deliverable

i.e., the workloops = the machine that builds the machine
- test frameworks
- access keys
- dev patterns
- briefs and lessons 

etc

it ensures that the mechanic who buildsout the solution has access to the tools they need in order to
1. verify the built the right thing
2. verify that what they built does the right thing

previously we launched the concept of 'playloops'. that was a start in this direction

lets deprecate that concept in favor of this 'workloop' blueprint concept.

i.e.,

3.3.0.blueprint.workloops = netnew, replaces enframe.playloops
3.3.1.blueprint.codepaths = renamed old, replaces 3.3.blueprint 

also, lets keep the playloops research, but instead, call it 'research.frames.testloops' and focus it on the test frameworks 

and lets also add a distill.repros.experience and focus it on a distillation of reproductions that will replicate the end user experience of the product.
