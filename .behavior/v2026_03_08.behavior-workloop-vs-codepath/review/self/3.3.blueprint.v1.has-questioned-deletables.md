# self-review: has-questioned-deletables

## components reviewed for deletion

### 1. can `3.2.enframe.playloops` be removed entirely?

**verdict**: yes, delete it.

- playloops tried to do too much (test frameworks + user journey repros)
- split into factory blueprint (frameworks) and repros.experience (journeys)
- if we had to add it back, we wouldn't — the split is cleaner

### 2. can the three-tier research prefix (3.1.1, 3.1.2, 3.1.3) be simplified?

**verdict**: keep it, but questioned.

- could we use just two tiers? (external vs internal)
- no — external.product vs external.factory is a meaningful distinction
- product = domain knowledge, factory = tools knowledge
- three tiers is the simplest that works

### 3. can `3.2.distill.repros.experience` be merged into the product blueprint?

**verdict**: no, keep separate.

- repros.experience is a distillation (synthesis of research)
- product blueprint is a design (how to build)
- different phases, different purposes
- if merged, product blueprint becomes bloated

### 4. can the factory blueprint be optional?

**verdict**: no, keep required.

- even simple behaviors benefit from explicit tool declaration
- forces thought about verification before code
- can be minimal/empty but should exist

### 5. can we avoid file renames and just use the new prefix in new behaviors?

**verdict**: no, rename for consistency.

- mixed names would confuse
- rename is mechanical (one-time cost)
- benefits all future behaviors

### 6. can the guards for factory blueprint be skipped?

**verdict**: questioned, but keep light guard.

- factory is foundational to product
- needs review to ensure tools are adequate
- light guard (not heavy) is sufficient

### 7. did we optimize a component that shouldn't exist?

**reviewed each component**:

| component | should exist? | why |
|-----------|---------------|-----|
| factory blueprint | yes | separates tools from code |
| product blueprint | yes | already extant, just renamed |
| repros.experience | yes | makes user journeys explicit |
| 3.1.1 prefix | yes | groups product domain research |
| 3.1.2 prefix | yes | groups factory tools research |
| 3.1.3 prefix | yes | groups internal patterns |

no optimized component that shouldn't exist.

## simplest version that works

the blueprint represents the simplest version:
- two blueprints (factory + product) instead of one blob
- three research prefixes (external.product, external.factory, internal)
- one new distill (repros.experience)
- one deprecation (playloops)

any simpler would lose the factory/product distinction or the research organization.
