The CLI orchestrators in cliRadioTaskPull and cliRadioTaskPush contain inline decode-friction logic to conditionally build input objects for downstream domain operations using spread syntax combined with &&:

...(condition !== null && { prop: value })

This requires readers to mentally simulate the resulting object shape based on null checks before understanding what is passed to radioTaskPullAll/radioTaskPush.
