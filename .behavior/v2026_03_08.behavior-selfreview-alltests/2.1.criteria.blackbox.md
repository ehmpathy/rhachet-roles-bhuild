# blackbox criteria: self-review for all tests

## usecase.1 = verification gate (5.3)

### episode: clone runs verification

```
given a clone at 5.3.verification stone
  when the clone runs `npm run test`
    then all tests execute (no skips)
    then test results are captured
      sothat the clone knows pass/fail status

  when all tests pass
    then the clone can proceed to mark stone as passed
      sothat verification gate opens

  when some tests fail
    given the clone can fix the failure
      then the clone fixes the tests
      then the clone retries `npm run test`
        sothat all tests pass

    given the clone cannot fix the failure (human-only blocker)
      then the clone emits 5.3.verification.handoff.v$N.to_human.md
      then the handoff explains what was tried
      then the handoff explains why human intervention is required
      then the handoff ends with rewind instruction
        sothat the human knows exactly what to do
```

### episode: behavior coverage check

```
given a clone at 5.3.verification stone
  when the clone reviews 0.wish.md and 1.vision.md
    then the clone identifies all behaviors promised
      sothat coverage can be verified

  when a behavior has an acceptance test
    then the clone notes the test file
    then the clone confirms the test runs (no skip)
      sothat coverage is proven

  when a behavior lacks an acceptance test
    then the clone writes the acceptance test
    then the clone runs the test
      sothat coverage gap is closed

  when a behavior cannot be tested (human-only blocker)
    then the clone emits handoff with what human needs to unblock
      sothat the human can enable the test
```

### episode: human unblocks and rewinds

```
given a human receives 5.3.verification.handoff.v$N.to_human.md
  when the human reads the handoff
    then the human sees what was tried
    then the human sees why intervention is required
    then the human sees the rewind instruction
      sothat the human knows the path forward

  when the human unblocks (grants credentials, access, etc.)
    then the human runs `rhx route.stone.set --stone 5.3.verification --as rewound`
      sothat the route rewinds to verification gate

  when the clone retries verification
    then the previously blocked tests now pass
      sothat verification completes
```

## usecase.2 = playtest gate (5.5)

### episode: clone emits playtest

```
given a clone at 5.5.playtest stone
  when the clone reviews 0.wish.md and 1.vision.md
    then the clone identifies all behaviors to verify manually
      sothat playtest scope is clear

  when the clone writes 5.5.playtest.md
    then the playtest has step-by-step instructions
    then each step is clear and unambiguous
    then expected outcomes are explicit
    then edge cases are covered
      sothat human can follow without prior context
```

### episode: playtest quality self-review

```
given a clone has emitted 5.5.playtest.md
  when the clone self-reviews has-clear-instructions
    then each step can be followed by a human without context
    then commands are copy-pasteable
    then outcomes are explicit
      sothat clarity is verified

  when the clone self-reviews has-vision-coverage
    then every behavior from wish/vision has a verification step
      sothat no behavior is left untested manually

  when the clone self-reviews has-edgecase-coverage
    then edge cases are identified
    then edge cases have verification steps
      sothat unusual inputs and boundaries are covered
```

### episode: human runs playtest and approves

```
given a human receives 5.5.playtest.md
  when the human follows each step
    then the human verifies the deliverable works
    then the human catches any UX issues automated tests missed
      sothat manual QA is complete

  when the human is satisfied
    then the human runs `rhx route.stone.set --stone 5.5.playtest --as approved`
      sothat the playtest gate opens

  when the human finds issues
    then the human provides feedback
    then the route rewinds for fixes
      sothat issues are addressed before approval
```

## usecase.3 = zero tolerance enforcement

### exchange: skip detection

```
given test output from `npm run test`
  when output contains `.skip()` or `.only()`
    then verification fails
      sothat skips are not tolerated

  when output contains "skip if no credentials" pattern
    then verification fails
      sothat silent bypasses are caught

  when output contains prior failures carried forward
    then verification fails
      sothat legacy failures are not ignored
```

### exchange: flake detection

```
given test output from `npm run test`
  when a test passes inconsistently (flaky)
    then the clone must fix the flake
    then the clone cannot proceed until stable
      sothat zero flakes is enforced
```
