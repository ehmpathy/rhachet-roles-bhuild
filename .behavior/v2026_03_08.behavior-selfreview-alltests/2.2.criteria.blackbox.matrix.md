# blackbox criteria matrix: self-review for all tests

## matrix 1: test execution outcomes (5.3)

| ind: test result | ind: clone can fix | dep: action | dep: artifact |
|------------------|-------------------|-------------|---------------|
| all pass | n/a | proceed to mark stone | none |
| some fail | yes | fix and retry | none |
| some fail | no (human blocker) | emit handoff | 5.3.verification.handoff.v$N.to_human.md |

## matrix 2: behavior coverage (5.3)

| ind: has acceptance test | ind: can test | dep: action | dep: artifact |
|--------------------------|---------------|-------------|---------------|
| yes | yes | note file, confirm runs | none |
| no | yes | write test, run test | new acceptance test |
| no | no (human blocker) | emit handoff | 5.3.verification.handoff.v$N.to_human.md |

note: "yes + no (human blocker)" is not enumerated because if a test extant, it can run. blocker only applies when test must be written but cannot be.

## matrix 3: human unblock flow (5.3)

| ind: human action | dep: route state | dep: next step |
|-------------------|------------------|----------------|
| unblocks | rewound | clone retries verification |
| does not unblock | blocked | route remains at 5.3 |

## matrix 4: playtest quality self-reviews (5.5)

| ind: self-review slug | dep: verification target |
|-----------------------|--------------------------|
| has-clear-instructions | steps followable, commands copy-pasteable, outcomes explicit |
| has-vision-coverage | every wish/vision behavior has verification step |
| has-edgecase-coverage | edge cases identified and covered |

## matrix 5: human playtest verdict (5.5)

| ind: human verdict | dep: action | dep: route state |
|--------------------|-------------|------------------|
| satisfied | approve | gate opens |
| finds issues | provide feedback | rewinds for fixes |

## matrix 6: zero tolerance patterns

| ind: test output pattern | dep: verification result | dep: rationale |
|--------------------------|--------------------------|----------------|
| contains .skip() | fails | skips hide problems |
| contains .only() | fails | partial runs mask gaps |
| contains "skip if no credentials" | fails | silent bypasses unacceptable |
| contains prior failures carried forward | fails | legacy failures not ignored |
| passes inconsistently (flaky) | blocked until stable | zero flakes enforced |

## gap analysis

all meaningful combinations are covered:

- **matrix 1**: complete — covers pass, fail-fixable, fail-blocker
- **matrix 2**: complete — covers has-test, no-test-can-write, no-test-blocker
- **matrix 3**: complete — covers unblock and no-unblock
- **matrix 4**: complete — all three self-review slugs specified
- **matrix 5**: complete — covers approve and reject
- **matrix 6**: complete — all zero-tolerance patterns enumerated

## decomposition notes

dimensions are minimal (2-3 independent variables per matrix). no decomposition needed — each matrix is focused on a single concern:

1. test execution outcomes
2. behavior coverage
3. human unblock flow
4. playtest quality
5. human verdict
6. zero tolerance patterns

the separation into 6 matrices reflects the natural boundaries of the behavior.
