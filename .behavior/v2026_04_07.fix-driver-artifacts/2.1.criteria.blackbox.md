# blackbox criteria: yield.md artifact convention

## usecase.1 = behaver follows stone instructions

```
given(a behaver is on a stone that emits an artifact)
  when(they read the stone instructions)
    then(the emit target is `{stone-prefix}.yield.md`)
      sothat(they know exactly where to write the output)
    then(the name is self-documented — "yield" means output of the stone)
      sothat(they understand the artifact's purpose without explanation)

given(a behaver references prior stone outputs)
  when(they read stone instructions that reference other artifacts)
    then(references use `*.yield.md` pattern)
      sothat(they can find the referenced files easily)
    then(referenced files sort adjacent to their source stones)
      sothat(navigation is intuitive in file browsers)
```

## usecase.2 = behaver reviews behavior directory

```
given(a behaver opens a behavior directory in a file browser)
  when(files are sorted alphabetically)
    then(each stone file is followed by its yield file)
      sothat(related files are grouped together)
    then(the pattern is consistent: `{prefix}.stone` then `{prefix}.yield.md`)
      sothat(the structure is predictable and learnable)

given(a behaver wants to find all stone outputs)
  when(they glob for `*.yield.md`)
    then(all artifact outputs are returned)
      sothat(discovery is straightforward)
```

## usecase.3 = behaver creates new behavior

```
given(a behaver initializes a new behavior directory)
  when(templates are copied)
    then(all emit targets use `yield.md` convention)
      sothat(new behaviors follow the updated pattern)
    then(all references use `yield.md` convention)
      sothat(cross-stone references are consistent)
```

## exchange experience

### input: stone template content

| field | before | after |
|-------|--------|-------|
| emit target | `$BEHAVIOR_DIR_REL/{stone}.v1.i1.md` | `$BEHAVIOR_DIR_REL/{stone}.yield.md` |
| references | `$BEHAVIOR_DIR_REL/*.v1.i1.md` | `$BEHAVIOR_DIR_REL/*.yield.md` |

### output: behavior directory structure

| before | after |
|--------|-------|
| `3.3.1.blueprint.product.v1.i1.md` | `3.3.1.blueprint.product.v1.yield.md` |
| `1.vision.md` (no change for stones without version suffix) | `1.vision.yield.md` |

### errors: none

this is a template content change — no runtime errors possible.

## boundary conditions

| condition | behavior |
|-----------|----------|
| stone has no emit target | no change (some stones are guard-only) |
| stone has version suffix (`v1`) | yield follows: `{stone}.v1.yield.md` |
| stone has no version suffix | yield follows: `{stone}.yield.md` |
| wildcard references | pattern preserved: `*.yield.md` |
