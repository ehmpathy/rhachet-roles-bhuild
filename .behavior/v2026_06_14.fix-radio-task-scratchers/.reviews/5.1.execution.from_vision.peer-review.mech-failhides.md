# blocker.1: failhide via try/catch that catches and hides BadRequestError

**rule**: .agent/repo=ehmpathy/role=mechanic/briefs/practices/code.prod/pitofsuccess.errors/rule.forbid.failhide.md.pt2.md

**locations**:
- src/domain.operations/radio/cli/getOneRadioTaskRepoFromCliArg.ts:45

The code uses try/catch to catch BadRequestError thrown by asRadioTaskRepoFromArg, converts it to a plain {error: {message, hint}} object, and returns it instead of rethrowing the error. Per the rule, errors that are thrown should not be caught and hidden; they should be rethrown (except for very restricted allowlisted control flow, but here the error class and context are lost). This is a failhide hazard as it silently converts the real error into non-error result data, losing stack traces and observability. The catch should rethrow or use proper error propagation.

**snippet**:
```ts
  } catch (error) {
    if (error instanceof BadRequestError)
      return {
        error: {
          message: error.message,
          hint: String(error.metadata?.hint ?? ''),
        },
      };
    throw error;
  }
```

# nitpick.1: manual exit codes instead of error classes with built-in semantics

**rule**: .agent/repo=ehmpathy/role=mechanic/briefs/practices/code.prod/pitofsuccess.errors/rule.require.exit-code-semantics.md

**locations**:
- src/contract/cli/radioTaskPull.ts:86
- src/contract/cli/radioTaskPush.ts:112
- src/domain.operations/radio/cli/getOneRadioTaskRepoFromCliArg.ts:20

The code manually invokes process.exit(2) for all constraint cases and uses custom result objects with error info. The rule specifies using ConstraintError/MalfunctionError (or BadRequestError/UnexpectedCodePathError) which have error.code.exit built-in (2 for constraint, 1 for malfunction). No need to set exit code manually. This is a nitpick because it bypasses the built-in semantics, risking inconsistency and missing automated retry decisions based on exit codes.

**snippet**:
```ts
    outputRadioResult({
      message: repoResult.error.message,
      isError: true,
      hint: { ask: '', command: repoResult.error.hint },
    });
    process.exit(2);
```
