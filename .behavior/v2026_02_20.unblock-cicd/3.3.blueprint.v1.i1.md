# blueprint: skip gh.issues radio tests until keyrack ready

## summary

the `BHUILD_DEMO_REPO_ACCESS_GITHUB_TOKEN` has been revoked. hardcode `describe.skip` on all gh.issues tests until keyrack provides a new token. the os-fileops tests remain unaffected.

## affected tests

| file | type |
|------|------|
| `src/access/daos/daoRadioTask/daoRadioTaskViaGhIssues.integration.test.ts` | integration |
| `blackbox/role=dispatcher/skill.radio.task.push.via-gh-issues.acceptance.test.ts` | acceptance |
| `blackbox/role=dispatcher/skill.radio.task.pull.via-gh-issues.acceptance.test.ts` | acceptance |

## approach

replace `describe` with `describe.skip` to hardcode skip until keyrack ready.

**current pattern:**
```ts
describe('daoRadioTaskViaGhIssues', () => {
  // tests...
});
```

**proposed pattern:**
```ts
// TODO: unskip once keyrack provides BHUILD_DEMO_REPO_ACCESS_GITHUB_TOKEN
describe.skip('daoRadioTaskViaGhIssues', () => {
  // tests...
});
```

this pattern:
- skips the entire suite unconditionally
- preserves all test logic unchanged
- clear TODO marks the temporary state
