# blackbox criteria: init.behavior --name @branch

## usecase.1 = create behavior from feature branch

```
given('on a feature branch with author prefix')
  # e.g., casey/add-user-auth
  when('user runs: rhx init.behavior --name @branch')
    then('behavior created with last segment as name')
      # .behavior/v{date}.add-user-auth/
      sothat('user does not need to type the branch name manually')
    then('standard behavior files scaffolded')
      sothat('user can proceed with behavior-driven development')
```

## usecase.2 = create behavior from multi-segment branch

```
given('on a branch with multiple segments')
  # e.g., org/team/cool-feature
  when('user runs: rhx init.behavior --name @branch')
    then('behavior created with last segment only')
      # .behavior/v{date}.cool-feature/
      sothat('prefixes are stripped regardless of depth')
```

## usecase.3 = create behavior from bare branch

```
given('on a branch without any slashes')
  # e.g., fix-urgent-bug
  when('user runs: rhx init.behavior --name @branch')
    then('behavior created with full branch name')
      # .behavior/v{date}.fix-urgent-bug/
      sothat('bare branch names work without modification')
```

## usecase.4 = reject protected branches

```
given('on main branch')
  when('user runs: rhx init.behavior --name @branch')
    then('error: cannot init behavior on protected branch')
      sothat('users do not accidentally create behaviors on main')

given('on master branch')
  when('user runs: rhx init.behavior --name @branch')
    then('error: cannot init behavior on protected branch')
      sothat('users do not accidentally create behaviors on master')
```

## usecase.5 = reject detached HEAD

```
given('in detached HEAD state')
  when('user runs: rhx init.behavior --name @branch')
    then('error: cannot expand @branch in detached HEAD state')
      sothat('users know they need to be on a branch')
```

## usecase.6 = case preservation

```
given('on a branch with mixed case')
  # e.g., Casey/Add-User-Auth
  when('user runs: rhx init.behavior --name @branch')
    then('behavior created with case preserved')
      # .behavior/v{date}.Add-User-Auth/
      sothat('user input is not mangled')
```

## exchange.1 = @branch token expansion

```
input:
  --name @branch

expansion:
  1. get current branch name via git
  2. if branch contains '/', take last segment
  3. if no '/', use full name
  4. preserve case

output:
  - expanded behavior name (string)
  - or error if protected branch or detached HEAD
```

## exchange.2 = error conditions

```
| condition | error message |
|-----------|---------------|
| on main | "cannot init behavior on protected branch: main" |
| on master | "cannot init behavior on protected branch: master" |
| detached HEAD | "cannot expand @branch in detached HEAD state" |
```
