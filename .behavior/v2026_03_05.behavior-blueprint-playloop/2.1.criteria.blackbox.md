# blackbox criteria: enframe.playloops

## episode experience

### usecase.1 = behaver discovers playloop framework via research

```
given(a behavior that needs a playloop framework)
  when(the behaver does not know what framework enables rapid playtest feedback)
    then(the behaver uses 3.1.research.playloop._.stone to discover options)
      sothat(they know what tools/patterns enable rapid playtest feedback)
    then(the research produces findings about playtest frameworks)
      sothat(the behaver can make an informed decision in enframe)

given(a behavior where the playloop framework is already known)
  when(the behaver knows what framework enables rapid playtest feedback)
    then(the behaver skips 3.1.research.playloop._.stone)
      sothat(they proceed directly to enframe.playloops)
```

### usecase.2 = behaver declares playloop framework via enframe

```
given(a behavior with playloop research complete or framework already known)
  when(the behaver reaches 3.2.enframe.playloops.stone)
    then(the behaver declares the playloop framework)
      sothat(builders know exactly how to reproduce the user experience)
    then(the playloop covers happy paths)
      sothat(the expected user journey is testable)
    then(the playloop covers guard paths)
      sothat(paths that keep users on track, self heal, or provide idempotency are testable)

given(a declared playloop)
  when(the behaver attempts to pass the stone)
    then(the guard self-review asks if it reproduces real user experience)
      sothat(lazy or unrealistic playloops are caught)
    then(the guard checks if it starts from user's entry point)
    then(the guard checks if it uses realistic inputs)
    then(the guard checks if it validates what user actually cares about)
```

### usecase.3 = builder leverages playloop in acceptance tests

```
given(a behavior with a declared playloop)
  when(the builder writes acceptance/experience tests)
    then(the tests follow the playloop framework)
      sothat(the user experience is reproduced in tests)
    then(the builder can rapidly iterate through the playloop)
      sothat(friction is caught before ship)

given(experience tests that leverage the playloop)
  when(the tests run)
    then(the builder sees the same experience the user will have)
      sothat(if it feels wrong in the test, they know it will feel wrong for the user)
```

## exchange experience

### stone: 3.1.research.playloop._.stone

```
given(a behavior that requires playloop framework research)
  when(the behaver invokes the research stone)
    then(the stone guides discovery of playtest frameworks)
    then(output is knowledge of what enables rapid playtest feedback)
    then(output informs the enframe.playloops stone)
```

### stone: 3.2.enframe.playloops.stone

```
given(a behavior ready for playloop declaration)
  when(the behaver invokes the enframe stone)
    then(the stone guides declaration of the playloop framework)
    then(output is a declared playloop that covers happy paths and guard paths)
    then(output enables builders to reproduce user experience in tests)

given(a playloop declaration)
  when(the behaver attempts to pass the guard)
    then(has-realworld-reproductions self-review is triggered)
    then(human approval is required)
```

### error boundaries

```
given(a playloop that does not start from user's entry point)
  when(the guard self-review runs)
    then(the review surfaces this gap)
      sothat(the behaver fixes it before approval)

given(a playloop that uses unrealistic inputs)
  when(the guard self-review runs)
    then(the review surfaces this gap)
      sothat(the playloop is corrected to use realistic inputs)

given(a playloop that validates the wrong things)
  when(the guard self-review runs)
    then(the review surfaces this gap)
      sothat(the playloop validates what users actually care about)
```
