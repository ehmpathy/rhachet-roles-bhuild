# blackbox criteria: minimalism & consistency self-reviews

## usecase.1 = behaver completes blueprint guard with minimalism reviews

```
given(behaver builds a blueprint for a behavior)
  when(blueprint guard triggers has-pruned-yagni review)
    then(behaver reviews each component for extras not prescribed)
      sothat(over-built code is removed or flagged as open question)
    then(behaver emits findings to reviews/{stone}.minimalism-yagni.md)

  when(blueprint guard triggers has-pruned-backcompat review)
    then(behaver reviews each backwards-compat concern)
    then(behaver flags unasked backwards compat as open question for wisher)
      sothat(wisher can confirm or deny the need)
    then(behaver emits findings to reviews/{stone}.minimalism-backcompat.md)
```

## usecase.2 = behaver completes blueprint guard with consistency reviews

```
given(behaver builds a blueprint for a behavior)
  when(blueprint guard triggers has-consistent-mechanisms review)
    then(behaver searches codebase for related extant mechanisms)
      sothat(duplicate mechanisms are discovered)
    then(behaver reviews each new mechanism against extant ones)
    then(behaver replaces duplicates with extant mechanisms or flags as open question)
    then(behaver emits findings to reviews/{stone}.consistency-mechanisms.md)

  when(blueprint guard triggers has-consistent-conventions review)
    then(behaver searches codebase for extant name conventions)
      sothat(divergent names are discovered)
    then(behaver reviews each name choice against extant conventions)
    then(behaver aligns divergent names with extant conventions or flags as open question)
    then(behaver emits findings to reviews/{stone}.consistency-conventions.md)
```

## usecase.5 = light guard includes all 4 reviews

```
given(a light blueprint guard template)
  when(behaver triggers the guard)
    then(guard includes has-pruned-yagni review)
    then(guard includes has-pruned-backcompat review)
    then(guard includes has-consistent-mechanisms review)
    then(guard includes has-consistent-conventions review)
      sothat(all behaviors get minimalism and consistency reviews)
```

## edge cases

```
given(behaver flags an open question for backcompat)
  when(wisher does not respond)
    then(behaver proceeds with elimination-as-default posture)
      sothat(unconfirmed backwards compat is not shipped)

given(behaver finds extant convention that seems wrong)
  when(alignment would perpetuate bad patterns)
    then(behaver flags as open question with suggested refactor scope)
      sothat(wisher can decide to refactor or accept)

given(no extant pattern exists in codebase)
  when(behaver introduces new pattern)
    then(behaver documents the precedent in findings)
      sothat(future code can follow the new convention)

given(a refactor task explicitly asks to change conventions)
  when(behaver reviews for consistency)
    then(consistency reviews relax for the scoped changes)
      sothat(intentional convention changes are not blocked)
```
