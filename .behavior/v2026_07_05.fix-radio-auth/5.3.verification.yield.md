### wisher-directed simplification (round 5)

the wisher removed the `suggestAsHuman` boolean from `genAuthFromKeyrack`. rationale: to thread a boolean through two functions purely to omit the `--auth as-human` line in **test env** was not worth the complexity — no real human reads a nudge in ci, and in real usage as-human is always a valid suggestion. the absent-key nudge now **always** offers both paths (set-the-key + as-human). the as-human ban in test env is unchanged (still enforced in getGithubTokenByAuthArg's as-human branch via `isTestEnv`); it is simply no longer pre-filtered out of the nudge text.

removed as newly-dead: genAuthFromKeyrack.test case6 (the one-path/omit-as-human case) and getGithubTokenByAuthArg.test case3-t2 (asserted the orchestrator passed `suggestAsHuman: false` in test env). unit count 32 → 30.
