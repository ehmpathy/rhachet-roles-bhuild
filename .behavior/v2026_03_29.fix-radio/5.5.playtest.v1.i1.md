# playtest: radio skill keyrack integration

byhand verification that the radio skill's keyrack auth integration works as designed.

---

## prerequisites

1. **keyrack unlocked with ehmpath credentials**
   ```bash
   rhx keyrack unlock --owner ehmpath --prikey ~/.ssh/ehmpath --env all
   ```

2. **npm dependencies installed**
   ```bash
   npm ci
   ```

3. **build complete**
   ```bash
   npm run build
   ```

4. **demo repo accessible**: `ehmpathy/rhachet-roles-bhuild-demo` (beaver app must be installed)

---

## sandbox

all file operations in this playtest target external resources (github issues), not local files.

---

## happy paths

### path 1: radio.task.push just works (keyrack default)

**action**:
```bash
npx rhachet run --skill radio.task.push -- --via gh.issues --repo ehmpathy/rhachet-roles-bhuild-demo --title "playtest task $(date +%s)" --description "byhand playtest verification"
```

**expected outcome**:
- exit code 0
- output shows "created" confirmation
- output shows "exid: {number}"
- output shows "repo: ehmpathy/rhachet-roles-bhuild-demo"
- output shows "via: gh.issues"
- no --auth flag was required

**pass criteria**: task is created without explicit --auth flag. keyrack provided the token transparently.

### path 2: explicit --auth env override

**action**:
```bash
export PLAYTEST_TOKEN=$(rhx keyrack get --owner ehmpath --env prep --key EHMPATH_BEAVER_GITHUB_TOKEN --value)
npx rhachet run --skill radio.task.push -- --via gh.issues --auth "as-robot:env(PLAYTEST_TOKEN)" --repo ehmpathy/rhachet-roles-bhuild-demo --title "explicit auth test $(date +%s)" --description "test explicit env auth"
unset PLAYTEST_TOKEN
```

**expected outcome**:
- exit code 0
- output shows "created" confirmation
- the explicit --auth mode was honored

**pass criteria**: explicit --auth overrides default keyrack path.

## edgey paths

### edge 1: keyrack not unlocked

**setup**: lock keyrack first (if possible) or test in a fresh shell without keyrack unlocked.

**action** (in fresh shell without keyrack):
```bash
# in a fresh terminal where keyrack is not unlocked
npx rhachet run --skill radio.task.push -- --via gh.issues --repo ehmpathy/rhachet-roles-bhuild-demo --title "should fail" --description "keyrack not unlocked test"
```

**expected behavior**:
- exit code non-zero
- error message contains "keyrack"
- error message includes actionable fix hint

**pass criteria**: clear error with remediation guidance, not a silent failure or cryptic error.

### edge 2: absent --repo flag

**action**:
```bash
npx rhachet run --skill radio.task.push -- --via gh.issues --title "no repo test" --description "test"
```

**expected behavior**:
- exit code 0
- defaults to current repo (context-aware default)
- output shows "repo: {current-repo}"

**pass criteria**: context-aware default, no error.

### edge 3: absent --title flag

**action**:
```bash
npx rhachet run --skill radio.task.push -- --via gh.issues --repo ehmpathy/rhachet-roles-bhuild-demo --description "no title"
```

**expected behavior**:
- exit code non-zero
- error message mentions title is required

**pass criteria**: helpful validation error.
