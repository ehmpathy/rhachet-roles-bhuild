# roadmap: fix-radio keyrack integration

execution plan for the fix-radio blueprint.

---

## phase 0: provision github app

**scope**: provision and apply the bhuild-beaver github app

**note**: the app declaration already exists at `provision/github.apps/resources.app.bhuild-beaver.ts`

**read first**:
- `provision/github.apps/readme.md` (provision walkthrough)
- `provision/github.apps/resources.app.bhuild-beaver.ts` (app declaration)

**checklist**:
- [ ] run declastruct plan: `GITHUB_TOKEN=... npx declastruct plan --wish ./provision/github.apps/resources.ts --into ./provision/github.apps/.temp/plan.json`
- [ ] review plan output
- [ ] run declastruct apply: `GITHUB_TOKEN=... npx declastruct apply --plan ./provision/github.apps/.temp/plan.json`
- [ ] store app credentials (appId, privateKey, installationId) in 1password as `github.app.bhuild-beaver`
- [ ] fill keyrack with app credentials: `rhx keyrack set --key BHUILD_BEAVER_GITHUB_TOKEN --env prep`

**acceptance criteria**:
```
given(bhuild-beaver app provisioned)
given(credentials stored in keyrack)
  when(keyrack unlock --owner ehmpath --env prep)
    then(BHUILD_BEAVER_GITHUB_TOKEN is available)
```

**verification**:
```sh
rhx keyrack unlock --owner ehmpath --env prep
rhx keyrack get --key BHUILD_BEAVER_GITHUB_TOKEN --env prep
# expect: status = granted
```

**depends on**: none

## phase 2: core

**scope**: `getGithubTokenByAuthArg` keyrack integration

**read first**:
- `.behavior/v2026_03_29.fix-radio/3.3.1.blueprint.product.v1.i1.md` (contracts section: getGithubTokenByAuthArg, getAuthFromKeyrack)
- `.behavior/v2026_03_29.fix-radio/3.1.3.research.internal.product.code.prod._.v1.i1.md` (if extant)

**checklist**:
- [ ] create `src/domain.operations/radio/auth/getAuthFromKeyrack.ts` with fail-fast error handling
- [ ] create `src/domain.operations/radio/auth/getAuthFromKeyrack.test.ts` with unit tests
- [ ] update `src/domain.operations/radio/auth/getGithubTokenByAuthArg.ts` to:
  - add `as-robot:via-keyrack(owner)` mode
  - set default to `as-robot:via-keyrack(ehmpath)` when `--auth` not specified

**acceptance criteria**:
```
given(getAuthFromKeyrack)
  when(keyrack returns status='granted')
    then(returns { token })
  when(keyrack returns status!='granted')
    then(throws with message + fix hint)

given(getGithubTokenByAuthArg)
  when(--auth not specified)
    then(defaults to as-robot:via-keyrack(ehmpath))
  when(--auth as-robot:via-keyrack(owner) specified)
    then(calls getAuthFromKeyrack with owner)
  when(--auth explicit mode specified)
    then(uses specified mode)
```

**verification**:
```sh
npm run test:unit -- src/domain.operations/radio/auth/getAuthFromKeyrack.test.ts
npm run test:integration -- src/domain.operations/radio/auth/getGithubTokenByAuthArg.integration.test.ts
```

**depends on**: phase 1 (keyrack.yml for integration tests)
