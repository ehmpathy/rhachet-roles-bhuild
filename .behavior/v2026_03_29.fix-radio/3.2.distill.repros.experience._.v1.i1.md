# distill: experience reproductions

user experience test reproductions for radio skill keyrack integration.

---

## experience reproductions

| experience | entry point | user actions | expected outcome | test type |
|------------|-------------|--------------|------------------|-----------|
| radio skill just works | `rhx radio.task.push` | run push, no --auth | task created in gh issues | acceptance |
| keyrack fill | `rhx keyrack fill` | fill creds once per machine | creds stored in vault | integration |
| repo init keys | `npx rhachet init --keys` | init once per repo | keyrack.yml linked | acceptance |
| env var fallback | `rhx radio.task.push` | no keyrack, GITHUB_TOKEN set | task created via env var | acceptance |
| clear error: not unlocked | `rhx radio.task.push` | keyrack not filled | error: "keyrack not unlocked" | acceptance |
| clear error: key absent | `rhx radio.task.push` | keyrack filled, key absent | error: "token not found in vault" | acceptance |

## journey test sketches

### journey 1: radio skill just works

```
given('[case1] developer has filled keyrack and initialized repo')
  when('[t0] before push')
    then('no issues exist with test title')
  when('[t1] push task with title and description')
    then('exits with code 0')
    then('output shows created confirmation')
    then('output shows exid')
    then('input/output matches snapshot')
```

#### step table

| step | action | user sees |
|------|--------|-----------|
| t0 | before push | clean state |
| t1 | `rhx radio.task.push --via gh.issues --title "test" --description "..."` | created: test, exid: 42, via: gh.issues |

#### t1 success case (snapshot target)

```bash
$ rhx radio.task.push --via gh.issues --title "test task" --description "automated test"

🎙️ created: test task
   ├─ exid: 42
   ├─ status: QUEUED
   ├─ repo: ehmpathy/rhachet-roles-bhuild-demo
   └─ via: gh.issues
```

### journey 2: env var fallback

```
given('[case1] ci/cd environment with GITHUB_TOKEN')
  when('[t0] before push')
    then('GITHUB_TOKEN is set')
    then('keyrack is not available')
  when('[t1] push task')
    then('exits with code 0')
    then('task created via env var token')
    then('input/output matches snapshot')
```

#### step table

| step | action | user sees |
|------|--------|-----------|
| t0 | verify env | GITHUB_TOKEN set, keyrack unavailable |
| t1 | `rhx radio.task.push --via gh.issues --title "ci task"` | created: ci task |

### journey 3: error experience - keyrack not filled

```
given('[case1] developer has not filled keyrack')
  when('[t0] attempt push')
    then('exits with non-zero code')
    then('output shows keyrack error')
    then('output shows remediation hint')
    then('input/output matches snapshot')
```

#### t0 error case (snapshot target)

```bash
$ rhx radio.task.push --via gh.issues --title "test"

⚠️ keyrack not unlocked for env=prep

hint: run `rhx keyrack fill --owner ehmpath --env prep`
```

## reproduction feasibility

### test utilities available

- `genConsumerRepo`: creates test repo with roles linked
- `runRhachetSkill`: invokes skill in consumer context
- `keyrack.source()`: loads test env credentials

### setup required

1. `.agent/keyrack.yml` declares EHMPATH_BEAVER_GITHUB_TOKEN
2. `jest.integration.env.ts` calls keyrack.source()
3. beaver app installed on demo repo

### concrete test sketch

```typescript
describe('radio.task.push via gh.issues', () => {
  const sharedRepo = useBeforeAll(async () =>
    genConsumerRepo({ prefix: 'radio-keyrack-' }),
  );

  given('[case1] keyrack provides auth', () => {
    when('[t0] push with title and description', () => {
      const result = useBeforeAll(async () =>
        runRhachetSkill({
          repo: 'bhuild',
          role: 'dispatcher',
          skill: 'radio.task.push',
          args: '--via gh.issues --repo ehmpathy/rhachet-roles-bhuild-demo --title "test" --description "auto"',
          repoDir: sharedRepo.repoDir,
        }),
      );

      then('exits with code 0', () => {
        expect(result.exitCode).toBe(0);
      });

      then('output matches snapshot', () => {
        expect(result.output).toMatchSnapshot();
      });
    });
  });
});
```
