# blueprint: fix-radio keyrack integration

blueprint to integrate keyrack into the radio skill auth flow.

---

## summary

integrate keyrack as the default auth mechanism for the radio skill:

1. `--auth` defaults to `as-robot:via-keyrack(ehmpath)` if not specified
2. add new `as-robot:via-keyrack(owner)` auth mode
3. keyrack failures fail fast with forwarded stdout + stderr

## contracts

### getGithubTokenByAuthArg (updated)

```typescript
export const getGithubTokenByAuthArg = async (
  input: { auth: string | undefined },
  context: {
    env: NodeJS.ProcessEnv;
    shx: (command: string) => Promise<{ stdout: string; stderr: string }>;
  },
): Promise<
  | { token: string; role: 'as-robot' }
  | { token: null; role: 'as-human' }
>
```

auth resolution:
- `--auth` defaults to `as-robot:via-keyrack(ehmpath)` if not specified
- supported modes:
  - `as-robot:shx(...)` → shell command
  - `as-robot:env(...)` → env var
  - `as-robot:via-keyrack(owner)` → keyrack unlock (NEW)
  - `as-human` → gh cli session

`as-robot:via-keyrack(owner)` behavior:
- calls getAuthFromKeyrack({ owner, env: 'prep', key: 'EHMPATH_BEAVER_GITHUB_TOKEN' })
- success → return token
- any error → **fail fast**, forward stdout + stderr

### getAuthFromKeyrack (new)

```typescript
const getAuthFromKeyrack = async (input: {
  owner: string;
  env: string;
  key: string;
}): Promise<{ token: string }>
// fail fast on any error:
// - status: 'absent'  → throw with message + fix hint
// - status: 'locked'  → throw with message + fix hint
// - status: 'blocked' → throw with reasons + fix hint
// - status: 'granted' → return { token: grant.key.secret }
```

implementation pattern:
```typescript
const attempt = await keyrack.get({ for: { key: input.key }, env: input.env, owner: input.owner });
if (attempt.status !== 'granted') throw new Error(`keyrack: ${attempt.message ?? attempt.reasons?.join(', ')}\n  fix: ${attempt.fix}`);
return { token: attempt.grant.key.secret };
```
