# blocker: execution stone 5.1

## what blocks me

### Blocker 1: CI GitHub Actions secret needs GitHub App JSON

**the EHMPATH_BEAVER_GITHUB_TOKEN secret must contain GitHub App JSON credentials, not an ephemeral token.**

current state:
- secret contains an expired `ghs_*` ephemeral token
- keyrack with mechanism `EPHEMERAL_VIA_GITHUB_APP` expects JSON to translate at runtime
- error: `status: blocked 🚫 — no value to validate`

fix required:
1. get the GitHub App JSON credentials:
   ```bash
   # get installation id
   gh api --method GET /orgs/ehmpathy/installations | jq '.installations[] | select(.app_slug == "beaver-by-bhuild") | {app_id, id}'

   # construct json with private key
   jq -c -n --rawfile key ~/path/to/beaver-by-bhuild.private-key.pem \
     '{appId: "APP_ID", privateKey: $key, installationId: "INSTALLATION_ID"}'
   ```

2. set the JSON as the GitHub Actions secret:
   ```bash
   echo '{"appId":"...","privateKey":"...","installationId":"..."}' | gh secret set EHMPATH_BEAVER_GITHUB_TOKEN
   ```

keyrack.source() in CI will then translate the JSON to an ephemeral `ghs_*` token at runtime.

### Blocker 2 (RESOLVED): rhachet daemon unlock translation

**Bug in rhachet: EPHEMERAL_VIA_GITHUB_APP mechanism not translated at unlock**

the credential is stored correctly in keyrack:
- keyrack get shows `status: granted`
- mechanism is `EPHEMERAL_VIA_GITHUB_APP`
- JSON blob contains `appId`, `privateKey`, `installationId`

but keyrack returns the **raw JSON blob** instead of a translated `ghs_*` token.

---

## root cause

in `rhachet/src/domain.operations/keyrack/session/unlockKeyrackKeys.ts` lines 200-256:

```typescript
// get secret from vault (raw JSON blob)
const secret = await adapter.get({ slug, ... });

// stores raw secret in daemon — NO TRANSLATION!
keysToUnlock.push(
  new KeyrackKeyGrant({
    key: { secret, grade },  // ← raw JSON, not ghs_* token
    source: { vault, mech },
    ...
  }),
);
```

the unlock flow reads the raw secret from the vault and stores it directly in the daemon **without a call to the mechanism adapter's `translate` method**.

for `EPHEMERAL_VIA_GITHUB_APP`, the mechanism adapter (`mechAdapterGithubApp`) has a `translate` method that converts JSON to `ghs_*` token, but the unlock flow never calls it.

---

## evidence

ran integration test that calls `keyrack.get()`:

```
token prefix: {"appId":"...
token length: 1766
starts with ghs_: false
```

the returned "token" is the raw JSON blob, not a GitHub installation access token.

---

## what needs to happen

**fix in rhachet** (not this repo):

`unlockKeyrackKeys.ts` should call `mechAdapter.translate()` for mechanisms that require it (like `EPHEMERAL_VIA_GITHUB_APP`) before it stores keys in the daemon.

example fix location:

```typescript
// after vault.get returns secret...
const secret = await adapter.get({ slug, ... });

// translate if mechanism requires it
const mechAdapter = context.mechAdapters[mech];
if (mechAdapter?.translate) {
  const translated = await mechAdapter.translate({ secret });
  // use translated.secret and translated.expiresAt
}
```
