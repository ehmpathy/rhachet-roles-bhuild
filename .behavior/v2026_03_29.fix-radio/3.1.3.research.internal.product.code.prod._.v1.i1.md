## 1. auth derivation patterns

### [EXTEND] getGithubTokenByAuthArg.ts

current implementation at `src/domain.operations/radio/auth/getGithubTokenByAuthArg.ts`:

```typescript
export const getGithubTokenByAuthArg = async (
  input: { auth: string | undefined },
  context: {
    env: NodeJS.ProcessEnv;
    shx: (command: string) => Promise<{ stdout: string; stderr: string }>;
  },
): Promise<
  | { token: string; role: 'as-robot' | 'env' }
  | { token: null; role: 'as-human' }
>
```

auth resolution order:
1. explicit `--auth as-robot:shx(...)` command
2. explicit `--auth as-robot:env(...)` env var
3. explicit `--auth as-human` for gh cli
4. fallback to `GITHUB_TOKEN` env var

**extension needed**: insert keyrack as first auth option before explicit flags.

proposed order:
1. [NEW] keyrack unlock for specific key
2. explicit `--auth` flag (if provided)
3. fallback to `GITHUB_TOKEN` env var

### [REUSE] auth return types

the return type structure is correct for keyrack integration:

```typescript
| { token: string; role: 'as-robot' | 'env' }
| { token: null; role: 'as-human' }
```

keyrack tokens would return `{ token: 'ghs_...', role: 'as-robot' }`.

## 2. cli entry point patterns

### [REUSE] radioTaskPush.ts

current implementation at `src/contract/cli/radioTaskPush.ts`:

```typescript
export const radioTaskPush = async () => {
  const { named } = getCliArgs({ schema: schemaOfArgs });

  // derive github auth
  const github = await getGithubTokenByAuthArg(
    { auth: named.auth },
    { env: process.env, shx },
  );

  // build context
  const context: ContextDispatchRadio = {
    github: { auth: github },
    // ...
  };

  // execute
  return pushRadioTask({ ... }, context);
};
```

this pattern is correct. changes to `getGithubTokenByAuthArg` will flow through automatically.

### [REUSE] context build pattern

the context build pattern separates auth resolution from skill execution. this is the right decomposition.

## 7. keyrack integration patterns

### [NEW] keyrack unlock for specific key

pattern from vision:

```bash
keyrack unlock --owner ehmpath --env prep --key EHMPATH_BEAVER_GITHUB_TOKEN
```

typescript equivalent:

```typescript
import { keyrack } from 'rhachet/keyrack';

const token = await keyrack.unlock({
  owner: 'ehmpath',
  env: 'prep',
  key: 'EHMPATH_BEAVER_GITHUB_TOKEN',
});
```

### [REUSE] test env keyrack.source pattern

from xai repo `jest.integration.env.ts`:

```typescript
import { keyrack } from 'rhachet/keyrack';

const keyrackYmlPath = join(process.cwd(), '.agent/keyrack.yml');
if (existsSync(keyrackYmlPath))
  keyrack.source({ env: 'test', owner: 'ehmpath', mode: 'strict' });
```

apply same pattern for radio skill tests.
