# blueprint: behavior sizes

---

## summary

add `--size nano|mini|medi|mega|giga` flag to `init.behavior` skill.

the flag controls which stone templates are included when a behavior is initialized. medi is the default. sizes are additive (each size includes all stones from smaller sizes). --size and --guard are orthogonal (both flags work independently).

// type derived from config keys - compiler enforces completeness
export type BehaviorSizeLevel = keyof typeof BEHAVIOR_SIZE_CONFIG;

// order derived from config - no separate array to maintain
const BEHAVIOR_SIZE_ORDER = (Object.keys(BEHAVIOR_SIZE_CONFIG) as BehaviorSizeLevel[])
  .sort((a, b) => BEHAVIOR_SIZE_CONFIG[a].order - BEHAVIOR_SIZE_CONFIG[b].order);

### subdirectory creation

// when template target path includes /, create parent directory
const targetDir = path.dirname(targetPath);
if (targetDir !== input.behaviorDir) {

### guard variants handle

the template filter must handle both:
1. size filter (is this template in the size level?)
2. guard filter (select .light or .heavy variant)

order: filter by size first, then select guard variant.
