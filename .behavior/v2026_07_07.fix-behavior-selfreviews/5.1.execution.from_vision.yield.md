## deterministic signal set (per window) — plan-mode backbone

- `editCount` — Edit/Write tool_use to ARTIFACT files in window (the `review/self/` articulation
  write is left out — else the rate saturates to ~100% by construction)
- `readReviewedArtifact` — Read of a reviewed artifact (not the articulation) in window
- `bashCount` — Bash tool_use (test runs; verification stones)
- `articulationChars` — assistant text volume in window (coarse proxy; the real critique is the
  articulation FILE captured by the experience below)
- window bounds: chained — `[prior-promise (or passed) → this-promise]`

## distilled experience (per window) — apply-mode enrichment

phase 1 distills `SelfReviewExperience` from the window's events, purely (no new inference):

- `articulation: string | null` — the `content` of the in-window Write to `<route>/review/self/…`
- `duration: number | null` — `lastEvent.timestamp − firstEvent.timestamp` (ms; null if absent)
- `files: SelfReviewFileTouch[]` — each ARTIFACT file touched (articulation left out):
  `{ path, mode: 'read' | 'write', diff: string | null }`
  - `diff` from the tool input: Write → `content`, Edit → `old→new`, Read → null
  - **bounded**: per-file diff capped (`…(truncated)`), file count capped (`+N more`) so 800+
    windows stay cheap for the cheap brain

phase 2 (`imagineSelfReviewVerdict`) grades the FULL experience — articulation text + per-file
diffs + duration — not the four scalars. determinism cannot tell whether the written critique was
real; that is the one call the brain makes.
