# blackbox criteria matrix: --wish @stdin|words

## dimensions extracted

### independent variables

1. **wish-mode**: how wish content is provided
   - `absent` — no --wish flag
   - `inline` — --wish "content"
   - `stdin` — --wish @stdin

2. **wish-content**: what content is provided
   - `empty` — "" or empty stdin
   - `non-empty` — actual content

3. **wish-file-state**: state of extant 0.wish.md
   - `absent` — file does not exist (new behavior)
   - `template` — file exists with template only (`wish = \n\n`)
   - `modified` — file exists with user content

4. **open-flag**: whether --open is provided
   - `absent` — no --open
   - `present` — --open <editor>

### dependent variables

1. **behavior-dir**: created | reused | not-created
2. **wish-file**: populated | template | unchanged | not-created
3. **editor**: opens | not-opened
4. **exit**: 0 | non-zero
5. **error**: none | content-required | already-modified | value-required

## decomposition analysis

### dimension count: 4

1. wish-mode
2. wish-content
3. wish-file-state
4. open-flag

### assessment

4 dimensions is at the threshold. however:
- wish-mode and wish-content are tightly coupled (content only matters if mode is inline/stdin)
- open-flag is orthogonal and composes cleanly
- wish-file-state is the only dimension that introduces conflict behavior

### recommendation

no decomposition needed. the matrices are:
- matrix 1: 13 rows — core contract
- matrix 2: 6 rows — editor interaction
- matrix 3: 8 rows — lifecycle

all reasonable sizes. the behavior is well-bounded.
