## playtest: --wish @stdin|words

### prerequisites

- rhachet-roles-bhuild package linked: `npx rhachet roles link --repo bhuild --role behaver`
- clean test directory: `rm -rf .temp/playtest-wish && mkdir -p .temp/playtest-wish`
- init temp git repo: `cd .temp/playtest-wish && git init && git commit --allow-empty -m "init" && git checkout -b feature/wish-test`

### sandbox

all operations target `.temp/playtest-wish/` — never pollutes the repo root

### happy paths

#### 1. inline wish content

**action**:
```sh
cd .temp/playtest-wish && rhx init.behavior --name happy-inline --wish "capture my thought inline"
```

**expected outcome**:
- exit code: 0
- stdout contains: `🦫 oh, behave!`
- stdout contains: `🍄 we'll remember,`
- file `.behavior/*happy-inline/0.wish.md` contains:
  ```
  wish =

  capture my thought inline
  ```

**acceptance test**: `skill.init.behavior.wish.acceptance.test.ts` case1

---

#### 2. stdin wish content (piped)

**action**:
```sh
cd .temp/playtest-wish && echo "piped content from stdin" | rhx init.behavior --name happy-stdin --wish @stdin
```

**expected outcome**:
- exit code: 0
- file `.behavior/*happy-stdin/0.wish.md` contains:
  ```
  wish =

  piped content from stdin
  ```

**acceptance test**: `skill.init.behavior.wish.acceptance.test.ts` case2

---

#### 3. combined with --open cat

**action**:
```sh
cd .temp/playtest-wish && rhx init.behavior --name happy-combined --wish "wish before open" --open cat
```

**expected outcome**:
- exit code: 0
- stdout contains: `wish before open` (cat outputs the wish file)
- stdout contains: `opened in cat`

**acceptance test**: `skill.init.behavior.wish.acceptance.test.ts` case6

---

#### 4. idempotent repeat (same content)

**action**:
```sh
cd .temp/playtest-wish && rhx init.behavior --name happy-inline --wish "capture my thought inline"
```

**expected outcome**:
- exit code: 0 (idempotent — same content passes)
- file unchanged

**acceptance test**: `skill.init.behavior.wish.acceptance.test.ts` case5

### edgey paths

#### 1. empty inline wish

**action**:
```sh
cd .temp/playtest-wish && rhx init.behavior --name edgy-empty --wish ""
```

**expected outcome**:
- exit code: non-zero
- stderr contains: `--wish must be a string`

**acceptance test**: `skill.init.behavior.wish.acceptance.test.ts` case3b

---

#### 2. whitespace-only stdin

**action**:
```sh
cd .temp/playtest-wish && echo "   " | rhx init.behavior --name edgy-whitespace --wish @stdin
```

**expected outcome**:
- exit code: 2
- stderr contains: `--wish requires content`

**acceptance test**: `skill.init.behavior.wish.acceptance.test.ts` case3

---

#### 3. conflict (different content)

**action**:
```sh
cd .temp/playtest-wish && rhx init.behavior --name happy-inline --wish "different content"
```

**expected outcome**:
- exit code: 2
- stderr contains: `wish file has been modified`
- stderr contains: `rm` (suggests how to overwrite)

**acceptance test**: `skill.init.behavior.wish.acceptance.test.ts` case4

---

#### 4. @branch expansion on protected branch

**action** (requires main branch):
```sh
cd .temp/playtest-wish && git checkout main && rhx init.behavior --name @branch --wish "test on main"
```

**expected outcome**:
- exit code: non-zero
- stderr contains: `cannot init behavior on protected branch: main`

**acceptance test**: `init.behavior.at-branch.acceptance.test.ts` case3

### pass/fail criteria

- ✓ pass if: all happy paths complete with expected outcomes, all edgey paths produce expected errors
- ✗ fail if: any step exits with unexpected code, any output differs from expected, any file content is incorrect
