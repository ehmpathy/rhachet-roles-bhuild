# research: internal prod code patterns

## pattern 1: @stdin handler

**[REUSE]**

### citation [1]: `src/contract/cli/feedback.take.set.ts` lines 51-57

```ts
// handle @stdin for response
let response: string;
if (named.response === '@stdin') {
  response = readFileSync(0, 'utf-8').trim();
} else {
  response = named.response;
}
```

### relevance

this is the exact pattern for `--wish @stdin|words`:
- check if value is `@stdin`
- if so, read from file descriptor 0 (stdin) via `readFileSync(0, 'utf-8')`
- otherwise, use the value directly

### note

the `.trim()` is important — strips tail newlines from heredocs.

## pattern 3: initBehaviorDir findsert semantics

**[REUSE]**

```ts
// findsert: skip if extant
if (existsSync(targetPath)) {
  kept.push(targetName);

```

### relevance

initBehaviorDir creates 0.wish.md from template if absent, keeps it if extant. this means:
- first run: creates `0.wish.md` with template content
- subsequent runs: keeps extant `0.wish.md`

our `--wish` logic runs AFTER initBehaviorDir, so the wish file will always exist when we check.

```ts
// after initBehaviorDir...

// if --wish provided, populate wish file
if (named.wish) {
  // get wish content
  let wishContent: string;
  if (named.wish === '@stdin') {
    wishContent = readFileSync(0, 'utf-8').trim();
  } else {
    wishContent = named.wish;
  }

  // validate non-empty
  if (!wishContent) {
    console.error('💥 error: --wish requires content');
    process.exit(1);
  }

  // check extant wish file
  const wishPath = join(behaviorDir, '0.wish.md');
  const wishCurrent = readFileSync(wishPath, 'utf-8');

  // if extant content is not template, error
  if (wishCurrent.trim() !== 'wish =') {
    console.error('💥 error: wish file has been modified');
    console.error('');
    console.error('to overwrite, delete the wish file first:');
    console.error(`  rm ${wishPathRel}`);
    process.exit(1);
  }

  // write populated wish file
  writeFileSync(wishPath, `wish =\n\n${wishContent}\n`);
}
```
