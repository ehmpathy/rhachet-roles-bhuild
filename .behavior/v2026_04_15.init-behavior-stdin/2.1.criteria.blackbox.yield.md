# blackbox criteria: --wish @stdin|words

## usecase.1 = inline wish capture

```
given(a new behavior name)
  when(user runs `rhx init.behavior --name foo --wish "my wish content"`)
    then(behavior directory is created)
      sothat(user has a structured place for their work)
    then(wish file contains `wish = \n\nmy wish content`)
      sothat(user's thought is captured without editor)
    then(route is bound to branch)
      sothat(hooks can drive the thoughtroute)
    then(output shows tree structure)
      sothat(user sees what was created)
```

## usecase.2 = stdin wish capture

```
given(a new behavior name)
  when(user pipes content via `echo "my wish" | rhx init.behavior --name foo --wish @stdin`)
    then(wish file contains `wish = \n\nmy wish`)
      sothat(user can pipe content from any source)
    then(behavior directory is created)
    then(output shows tree structure)

given(a new behavior name)
  when(user uses heredoc via `rhx init.behavior --name foo --wish @stdin <<'EOF'\nmultiline\ncontent\nEOF`)
    then(wish file contains `wish = \n\nmultiline\ncontent`)
      sothat(user can capture multiline wishes)
```

## usecase.3 = combined with --open

```
given(a new behavior name)
  when(user runs `rhx init.behavior --name foo --wish "quick note" --open nvim`)
    then(wish file contains `wish = \n\nquick note`)
      sothat(wish is pre-populated)
    then(editor opens with wish file)
      sothat(user can expand on their thought)
    then(wish content appears in editor)
      sothat(user sees their pre-populated note)
```

## usecase.4 = extant behavior reuse

```
given(an extant behavior directory with template-only wish file)
  when(user runs `rhx init.behavior --name foo --wish "new content"`)
    then(wish file is updated with `wish = \n\nnew content`)
      sothat(user can populate wish on subsequent run)
    then(behavior directory is reused)
      sothat(extant structure is preserved)

given(an extant behavior directory with user-modified wish file)
  when(user runs `rhx init.behavior --name foo --wish "different content"`)
    then(error is returned: "wish file has been modified")
      sothat(user's prior work is not silently overwritten)
    then(exit code is non-zero)
      sothat(caller can detect failure)
```

## usecase.5 = error: empty wish

```
given(a new behavior name)
  when(user runs `rhx init.behavior --name foo --wish ""`)
    then(error is returned: "wish content required")
      sothat(user knows to provide content)
    then(behavior directory is NOT created)
      sothat(incomplete state is avoided)
    then(exit code is non-zero)

given(empty stdin)
  when(user runs `echo "" | rhx init.behavior --name foo --wish @stdin`)
    then(error is returned: "wish content required")
      sothat(empty stdin is handled same as empty string)
```

## usecase.6 = error: absent value

```
given(a new behavior name)
  when(user runs `rhx init.behavior --name foo --wish`)
    then(error is returned: value required for --wish)
      sothat(user knows to provide content)
    then(behavior directory is NOT created)
```

## usecase.7 = backwards compatibility

```
given(a new behavior name)
  when(user runs `rhx init.behavior --name foo` without --wish)
    then(behavior directory is created)
    then(wish file contains template only: `wish = \n\n`)
      sothat(extant behavior is preserved)
    then(output shows tree structure)
```

## episode flow: quick capture

```
1. user has idea in terminal
2. user runs `rhx init.behavior --name my-idea --wish "capture this thought"`
3. behavior directory is created
4. wish file contains user's thought
5. user is done — no editor needed
```

## episode flow: capture then expand

```
1. user has idea in terminal
2. user runs `rhx init.behavior --name my-idea --wish "quick note" --open nvim`
3. behavior directory is created
4. wish file contains "quick note"
5. editor opens with wish file
6. user expands on their thought in editor
7. user saves and closes editor
8. user is done
```

## episode flow: multiline via heredoc

```
1. user has complex multi-point idea
2. user runs:
   rhx init.behavior --name complex-idea --wish @stdin <<'EOF'
   want to implement:
   - feature A
   - feature B
   - feature C
   EOF
3. behavior directory is created
4. wish file contains all points
5. user is done
```

## boundary conditions

| input | behavior |
|-------|----------|
| `--wish "content"` | inline content used |
| `--wish @stdin` | stdin content used |
| `--wish ""` | error: content required |
| `--wish` (no value) | error: value required |
| `--wish "x" --open nvim` | content used, then editor opens |
| no `--wish` flag | template only (backwards compat) |
| extant template wish | update with content |
| extant modified wish | error: already modified |
